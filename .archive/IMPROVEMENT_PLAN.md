# CTF Agent 改进计划

## 问题1：Flag检测不完善

### 当前问题
1. **检测模式单一**：只检测 `flag{...}` 和 `ctf{...}` 格式
2. **检测位置分散**：
   - `main.py:92` - 简单字符串匹配
   - `agent.py:331` - 状态更新检测
   - `agent.py:392` - JSON解析检测
3. **流程不终止**：检测到flag后，自动模式仍继续执行

### 改进方案

#### 1.1 增强Flag检测模式
**位置**：新建 `src/utils/flag_detector.py`

**检测规则**：
```python
# 标准格式
- flag{...}
- ctf{...}
- FLAG{...}
- CTF{...}

# 变体格式
rsctf{...}
ctft{...}
ctf_flag{...}

# Base64编码的flag
- 检测base64后包含flag关键字

# 十六进制编码
- 检测hex解码后包含flag

# 自定义格式（通过正则）
- 用户可配置的flag格式
```

#### 1.2 统一Flag检测入口
**位置**：`src/core/agent.py`

**改动**：
- 在 `run_step()` 方法末尾统一检测
- 检测到flag后设置 `self.flag_found = True`
- 返回时附加flag信息

#### 1.3 流程终止机制
**位置**：`main.py`

**改动**：
- 检查 `agent.flag_found` 状态
- 检测到flag后立即终止循环
- 显示flag并询问是否继续

## 问题2：日志不够清晰

### 当前问题
1. **格式混乱**：所有内容都是 `## ROLE\n\ncontent`
2. **无法区分**：
   - 看不出哪些是发给LLM的提示词
   - 看不出哪些是LLM的响应
   - 看不出哪些是工具执行结果
3. **缺少时间戳**：无法追踪执行顺序
4. **缺少结构**：没有步骤编号和层级

### 改进方案

#### 2.1 重构日志格式（纯文本格式）

- 使用纯ASCII字符，不依赖markdown渲染

**位置**：`src/core/agent.py`
